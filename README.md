# Lexical Analyzer in C

This project is a simple implementation of a lexical analyzer written in the C programming language. The program reads a C source file and breaks it into meaningful tokens such as keywords, identifiers, constants, operators, and symbols.

The main goal of this project is to understand how the lexical analysis phase of a compiler works and how source code is processed before syntax analysis.

---

## Project Overview

A lexical analyzer is the first stage of a compiler. Its job is to scan the source code character by character and convert the sequence of characters into tokens. These tokens are then used by the later phases of the compiler such as parsing and semantic analysis.

In this project, the lexical analyzer reads a C program from an input file and identifies different components of the code.

The analyzer is capable of detecting:

- Keywords (int, float, return, etc.)
- Identifiers (variable and function names)
- Integer constants
- Operators (+, -, *, /, =, etc.)
- Special symbols like braces and parentheses
- Preprocessor directives (#)
- Comments (single line and multi-line)
- Invalid or unknown characters

Each lexeme is printed along with the token type.

## Example

Example input inside `input.c`:

```c
#include <stdio.h>

int main() {
    int a = 5;
    int b = 10;
    float avg = (a + b) / 2;

    // print result
    printf("Average = %f", avg);

    return 0;
}
```

Example output generated by the lexical analyzer:

```
Lexeme          Token
-----------------------------
int             -> Keyword
main            -> Identifier
(               -> Open Parenthesis
)               -> Close Parenthesis
{               -> Open Brace
int             -> Keyword
a               -> Identifier
=               -> Operator
5               -> Integer Constant
;               -> End of Statement
```

## How to Run the Program

### Step 1: Compile the program

Use GCC to compile the program:

```
gcc lexical_analyzer.c -o lexical
```

### Step 2: Run the executable

```
./lexical
```

Make sure the `input.c` file is present in the same directory so the program can read it.
